# USport 信息架构与页面说明

文档版本：V1.0  
更新时间：2026-03-22  
适用范围：小程序端、移动端、未来运营后台

## 1. 设计目标

本文档用于定义：

- 不同端的职责边界
- MVP 页面清单
- 页面入口、出口、核心模块
- 关键业务路径

原则：

- 小程序负责“轻、快、转化”。
- 移动端负责“深、频、留存”。
- 管理后台负责“治理、运营、分析”。

## 2. 多端信息架构

### 2.1 小程序端

推荐底部 Tab：

1. 首页
2. 组局
3. 消息
4. 我的

- 登录与 onboarding
- 活动发现与报名

- 关键通知触达

### 2.2 移动端

推荐底部 Tab：

1. 发现
2. 组局

4. 我的

- 深度筛选
- 邀约关系沉淀
- 高频聊天

### 2.3 运营后台

推荐一级导航：

1. 用户中心
2. 活动中心
3. 举报工单

5. 运营配置
6. 数据看板

## 3. 小程序页面清单

### 3.1 `pages/login/login`

目标：

- 完成微信登录或手机号授权登录。

核心模块：

- 品牌介绍
- 微信授权按钮
- 手机号授权按钮
- 用户协议与隐私政策

出口：

- 登录成功进入首页
- 首次登录进入 onboarding

### 3.2 `pages/index/index`

目标：

- 承接活动发现和快速转化。

核心模块：

- 城市切换
- 运动筛选
- 推荐活动卡片
- 热门场馆
- 官方活动 Banner

出口：

- 活动详情
- 场馆详情
- 登录页

### 3.3 `pages/activity/detail`

目标：

- 展示活动信息并驱动报名。

核心模块：

- 活动信息
- 发起人信息
- 参与人数
- 费用说明
- 报名按钮
- 风险提示

出口：

- 报名成功页
- 聊天页
- 举报页

### 3.4 `pages/activity/create`

目标：

- 支持用户快速发起活动。

核心模块：

- 运动类型
- 时间地点
- 人数与费用
- 报名规则
- 可见性

### 3.5 `pages/messages/index`

目标：

- 查看会话列表和系统通知入口。

核心模块：

- 私聊会话
- 活动群聊
- 系统通知

### 3.6 `pages/user/user`

目标：

- 承接个人资料、我的活动、信用和设置。

核心模块：

- 头像昵称
- 资料完整度
- 我的邀约
- 我的活动
- 信用分
- 设置

### 3.7 `pages/user/profile-edit`

目标：

- 编辑基础资料和运动画像。

## 6. 关键业务路径

### 6.1 新用户路径

登录 -> 完善资料 -> 选择主运动 -> 浏览活动/搭子 -> 发起邀约或报名活动

### 6.2 搭子邀约路径

发现用户 -> 查看对方资料 -> 发起邀约 -> 对方处理 -> 建立会话 -> 线下成局 -> 评价/信用更新

### 6.3 组局路径

创建活动 -> 发布 -> 报名/审核 -> 成员聊天 -> 开场提醒 -> 签到 -> 完赛 -> 评价/举报

### 6.4 官方活动路径

运营后台创建 -> 小程序 Banner / 推荐位曝光 -> 用户报名 -> 到场 -> 数据回流

## 7. 页面级规则

### 7.1 权限规则

- 未登录用户只能浏览部分公开内容。
- 需要登录后才能发起邀约、创建活动、报名、发送消息。
- 高风险活动的发起需要更高等级认证。

### 7.2 状态提示规则

- 活动不可报名时必须明确提示原因。
- 邀约已过期、已拒绝、已接受要展示不同状态文案。
- 举报处理中和已结案应在“我的举报”有明确状态。

### 7.3 埋点建议

关键埋点：

- 登录成功
- 资料完善完成
- 发现页筛选使用
- 发起邀约
- 邀约接受
- 创建活动
- 报名活动
- 签到成功
- 举报提交
- 会员支付成功

## 8. 当前代码与目标 IA 的差距

当前仓库里：

- 小程序仅有 `index`、`login`、`user` 三页。
- 移动端仅有首页、登录页、个人页。
- 当前页面更像工程脚手架，不足以支撑 MVP。

因此建议后续页面开发顺序为：

1. 登录 / onboarding
2. 活动列表 / 活动详情
3. 创建活动
4. 我的活动 / 我的邀约
5. 消息与通知
6. 资料编辑 / 信用 / 会员

## 9. 结论

USport 的 IA 设计不应照搬传统内容社区，而应以“发现 -> 成局 -> 履约 -> 信任沉淀”为主线。只要页面结构围绕这条主线展开，多端产品就会更聚焦，也更适合 MVP 冷启动。
