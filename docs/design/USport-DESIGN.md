# USport 设计系统

文档版本：V1.0  
更新时间：2026-03-22  
方法参考：gstack `design-consultation` + `plan-design-review`

## 1. 设计定位

USport 不是泛社交，也不是传统工具型运动 App。

它的设计要同时表达三件事：

1. 真实运动的身体感
2. 同城成局的即时性
3. 平台履约与信任的可靠感

因此整体视觉方向定义为：

**Urban Court Energy**

关键词：

- 城市
- 动感
- 真实出汗
- 干净但不冷淡
- 有竞技张力但不压迫

## 2. 设计原则

### 2.1 成局优先

视觉设计首先服务于：

- 快速理解活动信息
- 快速判断是否适合自己
- 快速完成报名或邀约

### 2.2 信任优先

任何涉及：

- 线下见面
- 报名履约
- 举报安全

都必须比普通社交产品更强调信息透明、状态清晰、风险提示明确。

### 2.3 多端一致，节奏不同

小程序、移动端、后台必须共享同一套品牌语言，但节奏应不同：

- 小程序更轻、更亮、更直接
- 移动端更沉浸、更关系化
- 后台更克制、更高信息密度

### 2.4 避免 AI 平均脸

不做：

- 紫色渐变 + 白卡片的通用模板
- 纯工具型灰蓝后台感
- 泛社区信息流既视感

要做：

- 更像真实运动空间的材质感
- 更有场地氛围和时间感
- 更有“今天就能组起来”的行动冲动

## 3. 竞品启发

基于公开官方资料，设计上可借鉴三类产品的长处：

- [Playtomic](https://playtomic.com/) 的“找场、找局、快速加入”路径，适合 USport 的成局导向。
- [Strava](https://support.strava.com/hc/en-us/articles/221622188-Clubs-on-the-Mobile-App) 的俱乐部、活动、消息、路线等“围绕运动对象组织社交”的结构，适合关系沉淀。
- [Nike Run Club](https://www.nike.com/us/es/nrc-app/) 的陪伴感、激励感和运动情绪表达，适合品牌气质层。

由此得出的设计启发：

1. 首页不要做成纯内容流，而要做成“行动入口页”。
2. 活动卡片必须优先展示时间、地点、人数、门槛，而不是装饰。
3. 用户画像要偏“运动简历”而不是“社交名片”。
4. 成就、信用、履约这些信息要可见但不沉重。

## 4. 品牌气质

### 4.1 品牌人格

- 可靠
- 有速度感
- 年轻但不幼稚
- 有组织力
- 会鼓励人出门运动

### 4.2 品牌语气

文案语气建议：

- 简短
- 动作导向
- 鼓励式
- 不端着

示例：

- “今晚有人一起打吗？”
- “这场局还差 2 人”
- “你已经连续 3 周到场”
- “这场适合中级友好局”

## 5. 视觉系统

### 5.1 色彩系统

核心不是科技蓝，而是“场地绿 + 动能橙 + 夜训深色”。

#### 主色

- `Court Green` `#1D6B57`
- `Sprint Lime` `#C7F36B`

#### 强调色

- `Energy Orange` `#F46A2F`
- `Clay Red` `#C94F3D`

#### 中性色

- `Night Ink` `#111A1B`
- `Graphite` `#273234`
- `Fog` `#EAEFE9`
- `Paper` `#F7F6F1`

#### 功能色

- 成功 `#1F9D68`
- 警示 `#E4A72C`
- 危险 `#D64545`
- 信息 `#2F7CF6`

### 5.2 色彩使用策略

- 大面积背景优先 `Paper` 和浅雾灰，避免纯白过冷。
- 主 CTA 用 `Court Green` 或 `Energy Orange`。
- 活动状态、信用提示、风险提示必须有明确颜色层级。
- 不用荧光色做大面积底色，只在徽标、人数、标签等位置做动能点缀。

### 5.3 字体系统

建议的正式实现组合：

- 中文正文：`Alibaba PuHuiTi 3.0` 或 `MiSans`
- 英文数字标题：`DIN Condensed` / `Barlow Condensed`

如果前端落地时暂时无法引入品牌字体：

- 标题回退：`Bahnschrift`, `Arial Narrow`, sans-serif
- 正文回退：`PingFang SC`, `Microsoft YaHei`, sans-serif

### 5.4 字号层级

- Hero 数字：40 / 44
- 页面大标题：28 / 32
- 模块标题：20 / 24
- 卡片标题：16 / 18
- 正文：14 / 16
- 辅助说明：12 / 14
- 微型标签：11 / 12

### 5.5 圆角与阴影

- 页面主卡片圆角：20
- 二级卡片圆角：16
- 标签圆角：999
- 浮层圆角：24

阴影策略：

- 不做重阴影
- 用轻投影 + 边框 + 色块分层

## 6. 布局系统

### 6.1 小程序

- 页面节奏更紧凑
- 首屏必须出现一个强行动区
- 常用卡片采用上下叠层结构
- 大量使用半圆角胶囊标签和横向滚动筛选

### 6.2 移动端

- 支持更大留白和更强沉浸感
- 首页可采用“顶部英雄区 + 模块拼接”
- 用户与活动详情页要更重展示关系信息

### 6.3 后台

- 以效率优先
- 左侧导航 + 顶部状态栏 + 右侧内容区
- 强调状态筛选、批量处理、审计信息

## 7. 组件原则

### 7.1 活动卡片

必须包含：

- 运动类型
- 时间
- 地区
- 当前人数 / 上限
- 门槛标签
- 报名 CTA

### 7.2 用户运动画像卡

必须更像“运动名片”而不是“自拍卡”。

核心信息：

- 主运动
- 技能等级
- 常活动时段
- 常驻区域
- 到场记录 / 信用提示

### 7.3 信用与风险提示

原则：

- 明确
- 不吓人
- 可解释

例如：

- “已实名”
- “近 30 天 6 场全到”
- “首次见面建议选择公共场馆”

### 7.4 CTA 按钮

优先级：

- 一级：立即报名 / 发起邀约 / 创建活动
- 二级：收藏 / 分享 / 查看详情
- 风险型：取消 / 举报 / 拉黑

## 8. 动效原则

### 8.1 页面进入

- 轻微上浮
- 卡片分批出现

### 8.2 状态反馈

- 报名成功：轻量弹层 + 状态变更
- 签到成功：数字和徽章反馈
- 邀约接受：聊天入口或后续行动清晰出现

### 8.3 禁止项

- 过度弹跳
- 复杂粒子
- 泛电商式夸张庆祝

## 9. 多端差异化表达

### 9.1 小程序

视觉关键词：

- 轻快
- 高转化
- 直给

### 9.2 移动端

视觉关键词：

- 更成熟
- 更关系化
- 更适合高频使用

### 9.3 后台

视觉关键词：

- 清晰
- 可审计
- 稳定

## 10. 图像与插画建议

优先使用：

- 实景运动照片
- 场馆局部
- 城市场地纹理
- 路线、场地、人数、时间的图形化表达

避免：

- 过度二次元
- 伪 3D AI 人像
- 无具体场景的抽象装饰

## 11. 可访问性与适配

必须满足：

- 深浅色对比充足
- 核心按钮点击热区充分
- 小程序单手操作区域友好
- 主要状态不用颜色单独表达，要带图标或文字

## 12. 设计结论

USport 的 UI 不应长得像“运动版陌陌”，也不应长得像“场馆预订工具”。  
它应该让用户一眼感受到：

- 这是一个真的能组起局的平台
- 这里的人是来运动的
- 这里的关系是可信的
- 我今天就可以在这里找到下一场活动
