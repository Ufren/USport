# USport 高保真页面说明书

文档版本：V1.0  
更新时间：2026-03-22  
适用范围：微信小程序、移动端、运营后台

## 1. 文档目的

本说明书用于把现有 PRD、SRS、IA 和设计系统继续下沉为页面级高保真规格，确保：

- 产品、设计、前端、测试对页面理解一致
- 页面结构、视觉重点、状态展示、关键交互不产生歧义
- 多端共享同一产品逻辑，但保留不同节奏和呈现方式

本说明书不替代接口文档，也不替代交互原型文件。  
它是高保真页面真源。

## 2. 总体设计方向

### 2.1 统一品牌基调

多端统一采用：

- 关键词：城市、场地、动能、可信赖、今日可成局
- 主色：`Court Green`
- 强调色：`Sprint Lime`
- 行动色：`Energy Orange`

### 2.2 页面优先级原则

高保真优先服务：

1. 用户快速判断是否值得参与
2. 用户快速完成报名或邀约
3. 用户清楚知道状态和风险

### 2.3 页面密度差异

- 小程序：轻、快、首屏强转化
- 移动端：更沉浸、更适合高频浏览与关系沉淀
- 后台：高信息密度、低视觉噪音、强状态管理

## 3. 小程序端高保真页面

## 3.1 登录页

页面 ID：`MP-LOGIN`

### 目标

- 建立品牌第一印象
- 让用户快速完成登录
- 明确平台不是泛社交，而是运动成局平台

### 页面结构

1. 顶部品牌区
2. 中央价值主张
3. 场景化插画 / 运动照片区
4. 登录操作区
5. 协议说明区

### 视觉说明

- 背景采用浅米白 + 绿色动态渐层
- 中央主文案使用粗标题和压缩英文数字
- 插画或照片表现“球场、夜跑、城市灯光”氛围
- 登录按钮需足够醒目，避免被背景吞没

### 文案建议

- 主标题：`今晚就能组一场`
- 副标题：`找搭子、加入局、稳定出勤，把运动坚持下去`
- 按钮：`微信快捷登录`

### 核心组件

- 品牌标识
- 价值主张大标题
- 微信登录按钮
- 手机号快捷补充按钮
- 用户协议和隐私政策

### 页面状态

- 默认态
- 登录中
- 登录失败
- 协议未勾选

### 交互规则

- 首次点击登录先校验协议勾选
- 登录成功后：
  已有资料 -> 首页
  首次用户 -> onboarding

### 验收要点

- 首屏无需滚动即可完成登录
- 用户一眼能明白这是运动成局平台
- 错误提示不过度打扰但足够明确

## 3.2 首页

页面 ID：`MP-HOME`

### 目标

- 在最短时间内让用户看到“今天能参加什么”
- 让平台托底活动成为首屏最强锚点

### 页面结构

1. 城市条
2. 运动筛选胶囊
3. Hero 活动卡
4. 附近热门活动列表
5. 热门场馆横向区
6. 底部 TabBar

### 视觉说明

- Hero 卡使用深绿到青绿渐层
- 人数缺口和限时信息使用 `Sprint Lime`
- 二级活动卡使用白底高圆角，边缘轻阴影
- 场馆区可使用浅纹理和实景图增强真实感

### Hero 活动卡字段

- 活动标签：官方托底 / 热门 / 新手友好
- 活动标题
- 时间
- 地区
- 当前人数 / 上限
- 费用方式
- 立即报名 CTA

### 活动卡视觉层级

第一行：

- 运动类型标签
- 距离 / 是否今晚

第二行：

- 活动名称

第三行：

- 时间
- 区域
- 技能门槛

第四行：

- 到场率提示
- 剩余席位
- CTA

### 页面状态

- 默认推荐态
- 已筛选态
- 空列表态
- 定位失败态
- 未登录浏览态

### 空列表态文案

- 标题：`附近还没有合适的局`
- 说明：`试试切换运动、时间，或发起一场自己的局`
- CTA：`发起活动`

### 验收要点

- 首屏前两屏内必须可报名
- 城市和筛选切换响应清晰
- Hero 区强于普通列表，不可同质化

## 3.3 活动详情页

页面 ID：`MP-ACTIVITY-DETAIL`

### 目标

- 帮用户快速判断是否适合报名
- 降低首场活动的不确定感

### 页面结构

1. 顶部封面和状态标签
2. 关键信息区
3. 主办方信息区
4. 活动说明区
5. 适合人群 / 门槛标签
6. 风险提示区
7. 固定底部操作栏

### 关键信息区内容

- 标题
- 开始时间 / 结束时间
- 报名截止时间
- 地区和场地
- 当前人数 / 上限
- 费用说明

### 主办方区内容

- 头像
- 昵称
- 已实名状态
- 近 30 天到场率
- 近期活动数

### 风险提示区

适用于：

- 夜间活动
- 首次线下见面
- 私密活动

展示方式：

- 暖黄色柔和提示条
- 图标 + 2 行说明

### 底部固定栏

左侧：

- 可报名状态说明

右侧：

- 主 CTA：`立即报名`
- 不可报名时改为原因文案

### 页面状态

- 可报名
- 已报名
- 候补中
- 满员
- 已截止
- 已取消

### 验收要点

- 不可报名原因必须直接可见
- 核心信息不需要用户长时间滚动才能找到
- 风险提示不制造恐惧，但必须清楚

## 3.4 创建活动页

页面 ID：`MP-CREATE-ACTIVITY`

### 目标

- 帮用户快速、低门槛地发起一场局

### 页面结构

1. 页面引导说明
2. 运动类型选择
3. 时间与截止时间
4. 地点与区域
5. 人数与费用
6. 报名规则与可见性
7. 底部发布按钮

### 视觉说明

- 使用分段式表单卡片
- 每个卡片只容纳一类决策
- 必填项标识清晰
- 发布按钮固定底部

### 关键交互

- 时间选择后自动校验截止时间
- 人数变化联动候补建议
- 高风险条件触发额外提示

### 页面状态

- 草稿未完成
- 可发布
- 校验失败
- 发布中
- 发布成功

### 验收要点

- 表单不应让用户觉得像在填后台
- 每一屏可见字段有限，减少认知压力

## 3.5 我的页

页面 ID：`MP-ME`

### 目标

- 集中呈现用户资料、活动、信用、设置

### 页面结构

1. 顶部个人头图区
2. 资料完整度条
3. 我的活动入口卡
4. 我的邀约入口卡
5. 信用分与到场记录
6. 设置和帮助

### 视觉说明

- 头图区可采用浅绿色渐层和运动纹理背景
- 信用卡片不要做成冷冰冰的风控样式
- 强调“稳定到场”是积极资产

### 验收要点

- 用户能快速进入自己的核心事务
- 信用信息可见但不压迫

## 4. 移动端高保真页面

## 4.1 发现页

页面 ID：`APP-DISCOVER`

### 目标

- 同时承接活动与搭子发现
- 让移动端形成更强的浏览和复访动力

### 页面结构

1. 顶部城市与标题区
2. 发现类型切换
3. 推荐模块
4. 高级筛选条
5. 列表内容区
6. 底部导航

### 视觉说明

- 顶部使用深色英雄区，增强沉浸感
- 活动流和搭子流通过结构明显区分
- 推荐算法结果要有“为什么推荐给你”的解释性标签

### 页面状态

- 活动模式
- 搭子模式
- 高级筛选展开
- 无结果

### 验收要点

- 活动与搭子切换体验自然
- 推荐解释可理解，不神秘

## 4.2 搭子详情页

页面 ID：`APP-PARTNER-DETAIL`

### 目标

- 帮用户判断这个人是否值得发起邀约

### 页面结构

1. 顶部头像与身份信息
2. 运动画像模块
3. 常驻区域与时间模块
4. 履约与信用模块
5. 发起邀约按钮
6. 举报 / 拉黑入口

### 视觉说明

- 顶部头像区有更强人物感
- 运动画像用可视化标签而不是纯文本
- 履约记录用积极语言呈现

### 核心字段

- 主运动
- 技能等级
- 每周频次
- 常活动区域
- 常活动时段
- 到场率
- 近 30 天活动次数

### 验收要点

- 不像泛交友名片
- 更像“运动履历 + 约局适配度”

## 4.3 活动详情页

页面 ID：`APP-ACTIVITY-DETAIL`

- 成员预览
- 群聊入口

1. 封面与关键状态

## 6. 全局组件高保真规范

## 6.1 活动卡片

必须具备：

- 时间感
- 场地感
- 人数变化感
- 明确 CTA

卡片分三层：

1. 标签层
2. 核心信息层
3. 动作层

## 6.2 运动画像组件

展示顺序建议：

1. 主运动
2. 技能等级
3. 频次
4. 活动区域
5. 常用时间

## 6.3 状态标签

视觉要求：

- 同一状态全端颜色统一
- 不只靠颜色，还要有文案

## 6.4 固定底部操作条

适用页面：

- 活动详情
- 搭子详情
- 创建活动

原则：

- 只保留一个一级动作
- 其余操作降级

## 7. 页面级文案风格

### 7.1 要做

- 短句
- 明确动作
- 有运动语境

### 7.2 不要做

- 泛社交暧昧文案
- 过重营销口吻
- 术语化后台语言

## 8. 高保真页面验收标准

页面达到高保真的最低标准：

1. 视觉层级清楚
2. 核心 CTA 明确
3. 状态展示完整
4. 关键异常态已定义
5. 研发能依据文档实现，不需再次猜测

## 9. 结论

USport 的高保真页面不是为了“看起来高级”，而是为了把“发现、成局、履约、治理”四件事表达得足够清楚、足够可信、足够让人愿意立刻行动。
